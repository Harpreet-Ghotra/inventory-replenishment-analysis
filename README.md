## Project Overview

This project develops a data-driven inventory replenishment decision-support framework for a multi-product, multi-warehouse sales operation. Using the US Regional Sales Dataset (7,991 orders, 47 products, 6 warehouses, May 2018–Dec 2020), it answers the question:

> *How should inventory replenishment policies be adapted to demand variability and business cost objectives to minimise total cost while maintaining acceptable service levels?*

The analysis shows that safety stock is always cost-justified when stockout penalties (lost gross margin) are included, and that the optimal policy varies by product based on demand variability.

## Analysis Pipeline

The notebook runs a sequential 9-section workflow:

| Section | Description |
|---|---|
| 0 — Setup | Load and clean the dataset; parse dates, clean currency columns |
| 1 — EDA | Distributions, sales channel breakdown, monthly trends, lead time normality test |
| 2 — Demand Analysis | Aggregate monthly demand by product; compute CV; classify variability |
| 3 — Inventory Policy | Calculate Safety Stock, Reorder Point, and EOQ for each product |
| 4 — Monte Carlo | Simulate 500 demand scenarios per policy to test performance |
| 5 — Segmentation | Tailor policy to each product's CV (Low / Medium / High variability) |
| 6 — Full Cost Model | Add stockout penalty (lost gross margin) to complete the cost comparison |
| 7 — Sensitivity Analysis | Sweep service levels 80%–99.9% to find the cost-efficient zone |
| 8 — Backtest | Train on 2018–2019; test against actual 2020 demand |
| 9 — Recommendations | Final SS, ROP, EOQ per product with segment-appropriate policy |

## How to Run

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/mse433-inventory-analysis.git
cd mse433-inventory-analysis
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Launch Jupyter and run the notebook

```bash
jupyter notebook notebooks/MSE433_Supply_Chain_Analysis.ipynb
```

Run all cells in order from top to bottom (`Kernel → Restart & Run All`).

> **Note:** The notebook expects the CSV file at `../data/US_Regional_Sales_Data.csv`
> relative to the notebook location. If you open the notebook from a different directory,
> update the path in Section 0 accordingly.

### 5. Outputs

All Excel output files are saved to the `outputs/` folder automatically when the notebook runs.

## Assumptions

| Parameter | Value | Rationale |
|---|---|---|
| Holding cost rate | 25% of unit cost / year | Standard industry approximation |
| Ordering cost | $50 per order | Fixed assumption (no data available) |
| Service level target | 95% (default) | Standard inventory management benchmark |
| Stockout penalty | Gross margin per unit (Unit Price − Unit Cost) | Minimum economic cost of a stockout |

---

## Policy Segmentation

Products are assigned to policies based on demand variability (CV = Std / Mean):

| Segment | CV Range | Policy | Products |
|---|---|---|---|
| Low variability | CV < 0.45 | Simple SS – 95% | P-23, P-8, P-40 |
| Medium variability | 0.45 ≤ CV < 0.55 | Full SS – 95% | P-37, P-16, P-22, P-4, P-27, P-46, P-12, P-29, P-41, P-38 |
| High variability | CV ≥ 0.55 | Full SS – 97% | P-11, P-2 |
